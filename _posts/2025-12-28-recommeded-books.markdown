This is a list of books that I think bring value to any aspiring software engineer and software architect:

- Clean Code: Teaches you how to write clear, readable code that is easy for others to maintain. Don't apply absolutely everything and absolutely by the book. Always consider your context.
- Release It!: Shows how to build systems that stay running and handle the stress of real-world use. This one is gold for DevOps transformations and it's been published before DevOps was event a thing.
- Domain-Driven Design: Focuses on connecting your code directly to the core logic of the business you're serving. It's a dry book for sure and you need time to get into it. I would argue it is more like a reference book, but still useful to go thru once.
- The Mythical Man-Month: A classic that explains why managing people and timelines is often harder than the technical work itself. Projects that it touches upon will be alien to most readers nowadays, but here and there there are bits of information that are really useful.
- Software Architecture in Practice: Offers a practical look at how to make big-picture design choices that meet specific goals. Again, a reference book and dry but really useful to get the ilities in order.
- Designing Data-Intensive Applications: Explains the trade-offs behind modern tools like databases and distributed systems. Back to basics, but with relatively modern stack. You will read how things work, underneath your code :)
